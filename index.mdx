## What is this?

This project demonstrates production-ready multitenancy patterns for Java applications using Spring Boot 3.2 and Java 21. It provides three interconnected services that together show how to isolate data, authentication, caching, and AI queries on a per-tenant basis — with full observability via Grafana, Loki, Tempo, and Prometheus.

Two tenants are pre-configured — **dukes** and **beans** — each with their own Keycloak realm, PostgreSQL schema, and document collection.

## Key features

<CardGroup cols={2}>
  <Card title="Schema-per-tenant database" icon="database" href="/multitenancy/schema-per-tenant">
    Each tenant gets its own PostgreSQL schema, enforced by Hibernate's `MultiTenantConnectionProvider`.
  </Card>
  <Card title="Per-tenant OAuth2/OIDC" icon="shield-halved" href="/multitenancy/per-tenant-auth">
    Dynamic Keycloak realm selection so each tenant authenticates against its own identity provider.
  </Card>
  <Card title="Subdomain routing" icon="globe" href="/services/edge-service">
    Spring Cloud Gateway routes `dukes.rock` and `beans.rock` to the correct tenant context automatically.
  </Card>
  <Card title="Tenant-isolated cache" icon="bolt" href="/multitenancy/tenant-cache-isolation">
    Cache keys are prefixed with tenant ID to prevent cross-tenant data leakage.
  </Card>
  <Card title="Per-tenant vector store" icon="microchip-ai" href="/multitenancy/vector-store-filtering">
    ChromaDB metadata filtering ensures AI document retrieval is scoped to the requesting tenant.
  </Card>
  <Card title="Full observability" icon="chart-line" href="/observability/overview">
    Tenant ID flows through MDC logs, Micrometer traces, and Prometheus metrics via Grafana Alloy.
  </Card>
</CardGroup>

## Technology stack

| Component | Technology |
|-----------|-----------|
| Language | Java 21 (virtual threads enabled) |
| Framework | Spring Boot 3.2 |
| API Gateway | Spring Cloud Gateway 2023.0.1 |
| Authentication | Keycloak 24.0 (OIDC/OAuth2) |
| Database | PostgreSQL 16 with Hibernate multitenancy |
| Migrations | Flyway (per-tenant schema migrations) |
| AI/LLM | Spring AI + Ollama (llama2) |
| Vector Store | ChromaDB |
| Metrics | Micrometer + Prometheus |
| Tracing | OpenTelemetry + Grafana Tempo |
| Logging | Loki4j + Grafana Loki |
| Dashboards | Grafana OSS |
| Testing | Testcontainers |

## Project structure

```
spring-boot-multitenancy/
├── edge-service/        # API gateway: OAuth2 login, subdomain routing, token relay
├── instrument-service/  # REST API: schema-per-tenant data, per-tenant JWT validation
├── chat-service/        # AI service: RAG with per-tenant document filtering
├── platform/            # Keycloak, Grafana, Loki, Tempo, Prometheus configs
└── compose.yml          # Docker Compose for all backing services
```
